開始之前
  • 設定你是誰

    • $ git config -global user.name “YourName”

    $ git config --global user.email "YourEmail"

    或

    $ git config --global --edit

  • 檢視設定

    • $ git config --list

    也可以直接檢視 .git 資料夾下的 config

建立或取得Git Repository
  • 於當前位子建立 Repository

    • $ git init

  • 複製遠端 Repository 至本機

    • $ git clone [git-repository-url]

新增檔案
  • 將指定檔案add進stage

    • $ git add [filename]

    或可使用 * 字元加入所有檔案

    $ git add *

  • add錯檔案想取消（unstage）

    • $ git reset HEAD [filename]

    或

    $ git rm [filename] --cached

  • Commit

    • $ git commit -m "commit message"

  • 更改上一次的commit messgage

    • $ git commit --amend

  • 只有小修正不想重新commit

    • $ git add [file]

    $ git commit --amend -m "commit message"

    將這次修改並入上次commit

  • 查看Repository狀態

    • $ git status

    能查看目前位於哪個branch，未commit的更變檔案

遠端操作
  • 新增遠端節點

    • $ git remote add origin

  • 刪除遠端節點

    • $ git remote rm origin

  • 檢視遠端節點

    • $ git remote -v

  • 上傳

    • $ git push [origin master]

    此處是指定origin master分支

  • 上傳分支

    • $ git push origin [brach-name]

  • 刪除遠端某分支

    • $ git push origin :[branch-name]

  • 下載更新（PULL）

    • $ git pull [origin master]

    可指定特定分支

分支
  • 檢視目前分支

    • $ git branch

  • 建立分支

    • $ git branch [branch-name]

  • 刪除分支

    • git branch -d [branch-name]

  • 切換至某分支

    • $ git checkout [-b] [branch-name]

    若無該分支則建立並切換至該分支：-b

  • 合併分支

    • git merge [branch-name]

    要先 checkout 到要合併的主分支

  • 檢視已合併的分支

    • $ git branch --merged

特殊用途
  • GitIgnore

    • $ echo "node_modules" >> .gitignore

    在Repository的根目錄下新增檔案.gitigonre 檔案內容為要忽略的檔案名稱或資料夾，可以避免把一些測試用檔案或npm模組也放入Repository導致肥大。

  • 檢視Commit 紀錄

    • $ git log [--graph] [--oneline]

    檢視分支圖：--graph, 單行檢視：--oneline

  • Commit空資料夾

    • $ touch .gitkeep

    Git 預設無法commit空資料夾，在資料夾下新增.keep或.gitkeep檔案即可。

  • 新增標籤

    • $ tag tag [TagName]

  • 檢視標籤

    • $ git tag

  • 抓戰犯

    • $ git blame [FileName

  • 暫存

    • $ git stash save

  • 取出暫存

    • $ git stach apply

  • 刪除暫存

    • $ git stach drop
